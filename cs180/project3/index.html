A.3: Warp the Images

Actually warp the image (im2 = [x, y]) with respect to the image staying unwarpped as the projection dimension (im1 = [x', y']). First, tranform the four corner points of im2 to find the warpped range of im2 on the original projection dimension. Then, for each point in warpped range, inverse warping it back to im2 scale, to get the corresponding pixel color value, and fill back to warpped scale.

The results of inverse warping are float that are not exactly at the pixel points, therefore, we need to interpolate the points with a proper color value. In this part, I implement two interpolation methods:

  • Nearest Neighbor Interpolation: Round coordinates to the nearest pixel value
  • Bilinear Interpolation: Use weighted average of four neighboring pixels

A.4: Blend the Images into a Mosaic

Warp the images so they're registered and create an image mosaic with steps:

  1. Determin the canvas size, and warp both images into extended frame
  2. Center seam: calculate bwsidt (scipy.ndimage.distance_transform_edt) for both images and normalize to [0, 1] scale
  3. Create mask: Alpha = logical(dtrans1 > dtrans2)
  4. Implement multiresolution gussian blending from Project 2 on extended images with mask

Part B: FEATURE MATCHING for AUTOSTITCHING

B.1: Harris Corner Detection

In part A, we manually select corresponding points to recover homography matrix. In this part, our goal is to implement an algorithm automatically detect the corresponding points, using technique descibed in this paper. The ideally points should be at cormers or edges, which contains outstanding features that are easily to be matched with another image. To achieve this goal, we first use Harris Coner Detector extract corners and infer features of an image as interest points.

I use the bedroom images as examples:

Notice that the interest points detected by Harris are too many and clustered that almost cover whole image, therefore, we implement the Adaptive Non-Maximal Suppression (ANMS) to reduce the interest points from harris to 500 points that distributed evenly on the image.

B.2: Feature Descriptor Extraction

Now that we have enough points that distributed at every part of the images, and we want to match these points on two images in pairs. For each of the interest points, we setup a 40x40 window frame with the point in the center, and inside the window, extract a pixel for each 5 pixels step. The extracted pixels are formed as a 8x8 patch. This process is called Feature Descriptor Extraction.

From the 500 interest points, we randomly sampled 10 points with their feature descriptor patches and show them below, and their corresponding positions on the image.

B.3: Feature Matching

Now we have created a feature descriptor patch for each interest points, to match the features, we implement some Feature Matching steps:

  1. For each patch on im1, flatten the 8x8 matrix, and calculate its the Euclidean Distance with every patch on im2.
  2. Find the two nearest distances, and calculate their Lowe's ratio (1-NN / 2-NN). The first nearest distance is the potential matched point, a smaller Lowe's ratio means the first nearest distance is more unique.
  3. With a threshold consulted from Figure 6b in the paper, here I set threshold = 0.2. We reserve those points with Lowe's ratio less than the threshold, consider them as points on im1that have corresponding points.
  4. Reserve the 1-NN nearest distance points as the corresponding points on im2.

The reserved points are in pairs, an example is showed below.

B.4: RANSAC for Robust Homography

There are still some point pairs from last part that are not matched correctly, such as pair 7 and 21 from example. Therefore, we implement 4-point RANSAC (RANdom SAmple Consensus) algorithm to remove the outliers.

  1. Randomly select four pairs to compute homography matrix H (decribed in Part A.2).
  2. With the homography H, compute transformed points for all points in im2 that project to im1 plane, where p'2_i = Hp2_i.
  3. Find the inliners with dist(p'2_i, p1_i) less than ε. Here I set ε = 10.
  4. Count the number of inliners, keep the largest set of inliners, and repeat step 1-4 for 20 iterations.
  5. The largest set of inliners is the final denoised matched point sets.



Manunal vs. Automatic Stitching

With the final matched point pairs, I re-compute the mosaic blending images for my three examples. By comparing the mosaics stitched manually and automatically, the auto-stitching in the Mountain View amd Living Room examples have some misalignment that are fuzzy or unnatural transitive. The reason might be the elements are too complex, that the automative technique is not able to recognize the major objects in an image, and keeps some unimportant points, such as the clouds in Mountain View. The auto-stitching performs better in the Bedroom example, since the objiects are simple and edges are clear and straight.
